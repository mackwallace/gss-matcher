|---|---|
| Workflow Field ID | `intake-degree-goal` |
| Required | Yes |
| Input Type | Option cards (single select) |
| Validation | Exactly one of the three enum values |


### Field 8: Location Flexibility *(updated in v3 — two new regions)*
| Property | Value |
|---|---|
| Workflow Field ID | `intake-location-flexibility` |
| Required | Yes |
| Input Type | Radio buttons + conditional pickers |
| Used By | Agent 1 Scorer — Dimension 4: Location Match (prototype: flat 85 for all schools) |

**Type:** `string`

**Possible values:**
```
"Anywhere in US"
"Specific region — Northeast"
"Specific region — Mid-Atlantic"       ← new in v3
"Specific region — Southeast"
"Specific region — Midwest"
"Specific region — Southwest"
"Specific region — Mountain West"      ← new in v3
"Specific region — West Coast"
"Specific region — Pacific Northwest"
"Must stay local — [City], [State]"
```

**Region guidance:**
- `Mid-Atlantic` — covers MD, DC, VA, PA, NJ, NY, DE
- `Mountain West` — covers CO, UT, NV, ID, MT, WY

**Conditional behavior:**
- "Specific region" → shows region chip picker; combine as `"Specific region — [Region]"`
- "Must stay local" → reveals city + state text inputs; combine as `"Must stay local — Boston, MA"`

**Prototype note:** Location Match is a flat 85 in v0.1 regardless of value. Pass the full string for logging and future use.

## Agent Implementation Notes

1. **Research interests exact match:** Strings must exactly match `specialties` arrays in `gss_programs.json` (case-sensitive).
2. **Student status enum change:** Pass `"U.S. Citizen"` (not `"US Citizen / Permanent Resident"`). Agent 2 should handle TOEFL/visa context only when value is `"International Student"`.
3. **Two new regions:** `Mid-Atlantic` and `Mountain West` are now valid region values. KB files have not yet been reclassified — see `University KB/_KB_METADATA.md` for the pending audit.
4. **ResearchGate:** Treated identically to Scholar URL by Profile Builder. Pass `""` if not provided.
5. **Marketing consent:** Always pass `true` (form blocks submission otherwise). Use for downstream consent records.
6. **Empty optional fields:** Pass `""` for unprovided URL fields, `[]` for unprovided arrays. Do not omit keys.
7. **Session ID:** Always Worker-generated. Frontend never generates it.
